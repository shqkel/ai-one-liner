# PRD: AI One-Liner 크롬 익스텐션

## 1. 개요

브라우저에서 텍스트를 선택하면, 해당 문구/단어에 대한 **한 줄 설명을 툴팁으로 즉시 표시**하는 크롬 익스텐션이다. OpenAI Responses API(`gpt-4.1-mini`)의 웹 검색 기능을 활용하여 정확한 사실 기반 응답을 제공하며, 추가로 상세 설명 확장, 외부 AI 서비스 연동, 히스토리 네비게이션 등을 지원한다.

## 2. 핵심 기능

### F1. 텍스트 선택 → 한줄 설명 툴팁

- 사용자가 웹페이지에서 텍스트를 드래그로 선택하면, 선택 영역 근처에 툴팁을 표시한다.
- OpenAI API로 한 줄(1문장, max_output_tokens=60) 설명을 요청한다.
- 로딩 상태(스피너 + "설명을 가져오는 중...") 표시 후 결과를 렌더링한다.
- 최소 2글자 이상 선택해야 동작한다.
- 툴팁은 Shadow DOM으로 격리되어 호스트 페이지 스타일의 영향을 받지 않는다.
- 툴팁 위치가 뷰포트를 벗어날 경우 자동으로 재배치한다.
- 툴팁 외부를 클릭하면 닫힌다.

### F2. More (상세 설명)

- 한줄 설명 하단에 "More" 버튼을 제공한다.
- 클릭 시 동일 텍스트에 대해 3~4줄의 상세 설명(max_output_tokens=400)을 요청한다.
- 로딩 중에는 dot-loading 애니메이션(···)을 표시한다.
- 상세 설명은 구분선과 함께 한줄 설명 아래에 추가 표시된다.
- 이미 상세 설명이 로드된 엔트리에서는 More 버튼을 숨긴다.

### F3. AI 서비스 연동

- 툴팁 하단에 AI 서비스 버튼 그룹을 제공한다.
- 기본 트리거 버튼은 **ChatGPT**(OpenAI 아이콘)이며, 클릭 시 해당 서비스로 새 탭을 열어 선택 텍스트를 질문한다.
- 호버 시 나머지 4개 서비스 버튼이 슬라이드 애니메이션으로 펼쳐진다:
  - **Gemini** (`gemini.google.com`)
  - **Grok** (`grok.com`)
  - **Perplexity** (`perplexity.ai`)
  - **Claude** (`claude.ai`)
- 각 서비스의 SVG 아이콘을 인라인으로 포함한다.
- 새 탭 열기는 background service worker를 통해 `chrome.tabs.create`로 처리한다.

### F4. 히스토리 네비게이션

- 하나의 툴팁 세션 내에서 여러 텍스트를 검색한 이력을 배열로 관리한다.
- 히스토리가 2개 이상일 때 네비게이션 바(◀ ▶ 버튼 + 현재 위치/전체 수 표시)를 표시한다.
- 브라우저 히스토리와 동일한 방식으로, 현재 위치 이후의 forward 히스토리는 새 검색 시 삭제된다.
- 각 엔트리는 query, oneLiner, detailed, citations, detailedCitations, highlightText를 보존한다.
- 키보드 단축키: `Shift+좌우 방향키`로 텍스트 선택 후 자동 검색 (300ms 디바운스).

### F5. 트리거 모드 (자동/클릭)

- **자동 모드(auto)**: 텍스트 선택 즉시 툴팁이 표시된다. (기본값)
- **클릭 모드(manual)**: 텍스트 선택 시 트리거 아이콘(익스텐션 아이콘 18x18)만 표시하고, 아이콘 클릭 또는 `A` 키를 눌러야 검색이 실행된다.
  - 트리거 아이콘 호버 시 "A" 힌트 툴팁이 표시된다.
  - 트리거 아이콘 위치가 뷰포트를 벗어나면 자동 재배치한다.
- 모드 전환은 팝업의 토글 버튼으로 설정하며, `chrome.storage.sync`에 저장되어 실시간 반영된다.

### F6. 본문 텍스트 하이라이팅

- 자동 모드에서 텍스트 선택 시, 또는 클릭 모드에서 검색 실행 시 선택된 텍스트를 노란색(`#fef08a`) `<mark>` 요소로 래핑하여 하이라이트한다.
- DOM TreeWalker를 사용하여 Range 내의 텍스트 노드를 정확히 분할·래핑한다.
- 툴팁을 닫으면 하이라이트를 제거하고 원본 DOM을 복원(`normalize`)한다.
- 툴팁 내부의 한줄/상세 설명 텍스트에서도 검색어가 포함된 경우 `<mark>`로 강조 표시한다.

### F7. PDF 뷰어 대응 (컨텍스트 메뉴)

- Chrome PDF 뷰어 등 content script의 이벤트 감지가 불가능한 환경을 위해 컨텍스트 메뉴를 제공한다.
- 텍스트 선택 후 우클릭 → "AI One-Liner: \"%s\"" 메뉴를 선택하면 검색이 실행된다.
- background에서 API 호출을 수행한 후, content script에 `showOneLiner` 메시지를 전송한다.
- 결과는 뷰포트 중앙에 `position: fixed`로 표시한다.
- 팝업에 "?" 아이콘 호버 시 PDF 사용법 안내 말풍선을 표시한다.

### F8. API Key 미설정 안내

- API Key가 설정되지 않은 상태에서 텍스트를 선택하면, 툴팁에 "🔑 API Key가 필요합니다." 메시지와 "설정 열기" 버튼을 표시한다.
- "설정 열기" 클릭 시 `chrome.action.openPopup()`으로 팝업을 연다.
- background API 호출 시에도 키가 없으면 에러 메시지를 반환한다: "API Key가 설정되지 않았습니다. 익스텐션 아이콘을 클릭하여 API Key를 입력해주세요."

### F9. 인용 링크 표시

- OpenAI Responses API의 `url_citation` 어노테이션에서 인용 링크(title, url)를 추출한다.
- 중복 URL을 제거하고, 🔗 접두사와 함께 링크 목록을 툴팁에 표시한다.
- 응답 텍스트 내 인라인 인용 마크업(`[title](url)`, `([title](url))`)은 정규식으로 제거한다.
- More로 상세 설명을 로드한 경우 상세 설명의 인용으로 교체한다.
- 링크는 새 탭(`target="_blank"`)에서 열린다.

### F10. 키보드 단축키

| 키 | 동작 |
|---|---|
| `ESC` | 툴팁 닫기 + 트리거 아이콘 닫기 |
| `A` | 클릭 모드에서 트리거 아이콘이 표시된 상태일 때 검색 실행 (Ctrl/Meta/Alt 조합 제외) |
| `Shift+방향키` | 키보드로 텍스트 선택 시 자동/수동 모드에 맞춰 검색 트리거 (300ms 디바운스) |

### F11. 팝업 - API Key 관리

- 익스텐션 아이콘 클릭 시 팝업(300px 너비)에서 OpenAI API Key를 입력/저장/삭제할 수 있다.
- **미설정 상태**: 안내 문구 + API Key 발급 링크(`platform.openai.com/api-keys`) + 비밀번호 입력 필드 + 저장 버튼
- **설정 완료 상태**: "✅ API Key 설정됨" 상태 표시 + 마스킹된 키 표시(앞 4자 + **** + 뒤 4자) + 삭제 버튼
- API Key는 `chrome.storage.sync`에 `openai_api_key` 키로 저장된다.

### F12. 툴팁 내부 텍스트 선택 → 재검색

- 툴팁 내부의 설명 텍스트를 드래그로 선택(2글자 이상)하면, 같은 툴팁에서 새로운 검색을 수행한다.
- 이전 엔트리에 하이라이트 텍스트를 기록하여 돌아갔을 때 강조를 유지한다.
- 버튼이나 링크 위에서의 드래그는 무시한다.

## 3. 기술 스택

| 항목 | 선택 | 상세 |
|---|---|---|
| Manifest | V3 | Chrome Extension 최신 표준 |
| AI API | OpenAI Responses API | `gpt-4.1-mini` 모델, 웹 검색 도구(`web_search`) 활용 |
| Background | Service Worker | 컨텍스트 메뉴 등록, API 호출, 메시지 라우팅, 새 탭 열기 |
| Content Script | Vanilla JS (IIFE) | 모든 URL에 주입, Shadow DOM으로 UI 격리 |
| 스토리지 | `chrome.storage.sync` | API Key, 트리거 모드 동기화 저장 |
| UI | Vanilla JS + CSS | 외부 의존성 없이 경량 유지 |
| 권한 | `storage`, `activeTab`, `contextMenus` | 최소 권한 원칙 |
| 리소스 접근 | `web_accessible_resources` | 트리거 아이콘용 `icons/icon48.png` |

## 4. 사용자 플로우

### 4-1. 자동 모드 (기본)

```
1. 익스텐션 설치 → 팝업에서 API Key 입력
2. 웹페이지에서 텍스트 드래그 선택 (2글자 이상)
3. 선택 텍스트가 노란색으로 하이라이트됨
4. 선택 영역 아래에 툴팁 표시 (스피너 → 한줄 설명)
5a. "More" 클릭 → 3~4줄 상세 설명 추가 표시
5b. ChatGPT 아이콘 클릭 → ChatGPT 새 탭에서 해당 텍스트 질문
5c. AI 아이콘 호버 → 다른 AI 서비스(Gemini, Grok, Perplexity, Claude) 선택 가능
5d. 인용 링크 클릭 → 출처 페이지 새 탭 열기
6. 툴팁 내부 텍스트 선택 → 새로운 검색 (히스토리에 추가)
7. ◀/▶ 버튼으로 이전/다음 검색 결과 네비게이션
8. 툴팁 외부 클릭 또는 ESC → 툴팁 닫기 + 하이라이트 해제
```

### 4-2. 클릭 모드

```
1. 팝업에서 트리거를 "클릭"으로 변경
2. 웹페이지에서 텍스트 드래그 선택
3. 선택 영역 우측 상단에 트리거 아이콘(익스텐션 아이콘) 표시
4a. 트리거 아이콘 클릭 → 검색 실행
4b. A 키 입력 → 검색 실행
5. 선택 텍스트 하이라이트 + 툴팁 표시 (이후 자동 모드와 동일)
```

### 4-3. PDF 모드

```
1. PDF 파일을 Chrome에서 열기
2. PDF 내 텍스트 선택 후 우클릭
3. 컨텍스트 메뉴에서 "AI One-Liner: (선택 텍스트)" 클릭
4. 화면 중앙에 fixed 위치로 툴팁 표시
5. 이후 기능(More, AI 서비스, 인용 등)은 동일
```

## 5. 비기능 요구사항

- **응답 속도**: 한줄 설명 2초 이내 응답 목표 (gpt-4.1-mini + 웹 검색 기준)
- **범용성**: 모든 웹사이트에서 동작 (content script `<all_urls>` 전역 주입)
- **스타일 격리**: Shadow DOM을 사용하여 호스트 페이지의 CSS와 완전히 분리
- **경량**: 외부 라이브러리 의존성 없음, Vanilla JS/CSS만 사용
- **보안**: API Key는 `chrome.storage.sync`에 저장, 팝업에서 마스킹 표시
- **실시간 동기화**: 트리거 모드 변경 시 `storage.onChanged` 리스너로 content script에 즉시 반영
- **접근성**: 키보드 전용 사용자를 위한 `Shift+방향키` 텍스트 선택 지원
- **UX**: 뷰포트 경계 감지 후 툴팁/아이콘 자동 재배치, 부드러운 애니메이션(fadeIn, 슬라이드)

## 6. 제약 사항

- OpenAI API Key 필요 (사용자 직접 발급, `platform.openai.com/api-keys`)
- API 호출 비용은 사용자 부담
- Chrome 전용 (Manifest V3 기반)
- PDF 뷰어에서는 컨텍스트 메뉴 방식만 지원 (텍스트 선택 이벤트 감지 불가)
- `chrome.action.openPopup()`은 일부 Chrome 버전에서 미지원될 수 있음
- `surroundContents`로 하이라이트 적용 시 복잡한 DOM 구조에서는 실패할 수 있음 (try-catch로 처리)
- API Key는 `chrome.storage.sync`에 평문 저장 (Chrome 프로필 암호화에 의존)
